## 1. 系统概述

TA Platform 是一个面向高校助教工作的三端智能平台，支持**老师端**、**助教端**、**学生端**三种角色，集成 AI 问答、对话历史管理、实体机器人监控等功能。

### 技术架构

```
浏览器 / 手机PWA / 微信小程序
        │
        ▼
    Nginx (反向代理 + HTTPS)
        │
        ▼
    FastAPI (Python 后端)
        │
   ┌────┼────┐
   ▼    ▼    ▼
PostgreSQL  Redis  DeepSeek API
```

### 核心依赖

| 组件 | 用途 |
|------|------|
| FastAPI | 后端框架，提供 REST API + WebSocket |
| PostgreSQL 15 | 主数据库，存储用户/对话/机器人数据 |
| Redis 7 | 缓存、对话记忆、WebSocket 消息发布 |
| React 18 + TypeScript | 前端框架 |
| DeepSeek API | AI 大语言模型 |
| Docker Compose | 一键部署 |
| Nginx | 反向代理 + 静态文件 + HTTPS |

## 2. 功能清单

### 2.1 用户认证系统

| 功能 | 说明 |
|------|------|
| 注册 | 学号 + 手机号 + 密码 + 短信验证码 |
| 登录 | 学号或手机号 + 密码 |
| 角色分配 | 老师 / 助教 / 学生（注册默认为学生） |
| JWT 双令牌 | access_token（30分钟）+ refresh_token（7天） |
| 密码加密 | bcrypt 哈希存储 |

### 2.2 学生端功能

- AI 对话问答（基于 DeepSeek）
- 对话上下文记忆（服务端持久化，支持 Redis + PostgreSQL 双重存储）
- 文件上传提问（支持 PDF / PPTX / DOCX / 图片）
- 课程章节知识库问答（读取 PDF/TXT 课件内容）
- PPT 课件在线预览
- 语音输入（浏览器 Web Speech API）
- 对话历史管理（查看、切换、继续对话）
- 深色/浅色主题切换
- 移动端 PWA 支持（可安装到手机桌面）

### 2.3 老师端功能

- 学生列表（搜索、分页）
- 查看任意学生的**完整对话历史**（含时间、内容、AI 回答）
- 查看学生使用统计（消息数、对话数、登录次数、活跃时间）
- 仪表盘汇总（学生总数、消息总量、今日提问数等）
- 机器人状态监控（位置、电量、工作状态）
- 机器人提问历史（查看学生的语音提问和回答内容）

### 2.4 助教端功能

- 学生使用情况概览（仅次数和频率，**无法查看对话内容**）
- 仪表盘汇总（学生总数、消息总量统计）
- 机器人状态监控（位置、电量、工作状态）

### 2.5 机器人功能

- 实时状态上报（电量、位置坐标、工作状态）
- 语音问答（麦克风采集 → ASR 语音识别 → AI 回答 → TTS 语音合成 → 扬声器播放）
- 两轮差速驱动（前进、后退、转向）
- 网页端实时监控（所有角色可见）
- 模拟模式（无硬件时自动使用模拟数据）

## 3. 快速开始（本地开发）

### 3.1 环境要求

| 软件 | 版本要求 |
|------|---------|
| Python | 3.11+ |
| Node.js | 18+ |
| PostgreSQL | 15+ |
| Redis | 7+ |
| Docker（可选） | 24+ |

### 3.2 第一步：安装 PostgreSQL 数据库

PostgreSQL 是项目使用的数据库，用于存储用户、对话、机器人数据。

**Windows 安装：**

1. 打开浏览器，访问 https://www.postgresql.org/download/windows/
2. 点击 "Download the installer"，选择 **PostgreSQL 15.x** 版本
3. 下载完成后双击运行安装程序，一路点 Next
4. **关键步骤：设置密码**
   - 安装过程中会弹出设置 `postgres` 超级用户密码的窗口
   - 输入一个你记得住的密码（比如 `ta_platform`），**务必记住这个密码**
   - 这个密码就是后面 `.env` 文件中 `DB_PASSWORD` 的值
5. 端口保持默认 `5432`，点击 Next 直到安装完成
6. 安装完成后，在开始菜单找到 **pgAdmin 4**（PostgreSQL 的图形化管理工具），打开它

**创建数据库：**

在 pgAdmin 4 中操作：

1. 左侧浏览器树展开 `Servers` → `PostgreSQL 15`
2. 输入你安装时设置的密码连接
3. 右键点击 `Databases` → `Create` → `Database...`
4. Database 名称填：`ta_platform`
5. Owner 选择 `postgres`
6. 其他保持默认，点击 Save

这样数据库就创建好了。你也可以用命令行创建（在开始菜单找到 SQL Shell）：

```bash
# 在 SQL Shell (psql) 中输入：
CREATE DATABASE ta_platform;
# 然后输入 \q 退出
```

---

### 3.3 第二步：安装 Redis

Redis 是内存缓存数据库，用于对话记忆和消息队列。

**Windows 安装：**

Redis 官方不直接支持 Windows，但可以使用非官方版本：

1. 打开浏览器，访问 https://github.com/tporadowski/redis/releases
2. 下载最新的 `Redis-x64-xxx.msi` 文件
3. 双击安装，一路 Next，勾选 "Add Redis to PATH"
4. 安装完成后，Redis 会自动作为 Windows 服务运行
5. 验证：打开终端（CMD 或 PowerShell），输入：
   ```bash
   redis-cli ping
   # 如果返回 PONG，表示 Redis 正常运行
   ```

如果不想安装 Windows 版 Redis，也可以用项目自带的 Docker 方式启动（见 3.6 节），Docker 会自动启动 Redis。

---

### 3.4 第三步：编辑 .env 配置文件

`.env` 文件是项目的核心配置文件，包含所有密钥和数据库连接信息。它位于 `ta-platform/` 根目录下。

**创建 .env 文件：**

```bash
# 复制模板文件
cd ta-platform
copy .env.example .env    # Windows CMD
# 或 cp .env.example .env   # Git Bash / Linux / Mac
```

然后用**记事本、VS Code 或任意文本编辑器**打开 `.env` 文件，逐行修改。

**逐行配置详解：**

```ini
# ========== 数据库配置 ==========
# 这条是连接字符串，格式：数据库类型://用户名:密码@地址:端口/数据库名
# 用户名填 postgres，密码填你安装 PostgreSQL 时设置的密码
DATABASE_URL=postgresql+asyncpg://postgres:你在安装时设置的密码@localhost:5432/ta_platform

# 数据库密码（和上面连接字符串里的密码保持一致）
DB_PASSWORD=你在安装时设置的密码

# ========== Redis 配置 ==========
# 如果 Redis 在本地运行，不需要修改
REDIS_URL=redis://localhost:6379

# ========== JWT 密钥 ==========
# JWT 是登录令牌的加密密钥，随便生成一个64位的随机字符串即可
# 生成方法：打开浏览器搜 "random string generator 64"，或者用下面的命令生成
# Linux/Mac: openssl rand -hex 32
# Windows PowerShell: -join ((48..57)+(65..90)+(97..122) | Get-Random -Count 64 | % {[char]$_})
JWT_SECRET=把这里改成一串64位的随机字符，比如aB3xK9mP2qR7vY1wZ6nL4cF8oT5jE0hU3sG9aD6pM2wV5yB8nC1rX4zJ7kQ0tF9
JWT_ACCESS_EXPIRE_MINUTES=30
JWT_REFRESH_EXPIRE_DAYS=7

# ========== DeepSeek AI 配置 ==========
# ★★★ 最重要的一步：获取 DeepSeek API Key ★★★
# 1. 打开浏览器访问 https://platform.deepseek.com
# 2. 注册账号（用手机号即可）
# 3. 登录后在左侧菜单找到 "API Keys"
# 4. 点击 "创建 API Key"，命名为 "ta-platform"
# 5. 复制生成的 key（格式：sk-xxxxxxxxxxxxxxxxxxxxxxxx）
# 6. 粘贴到下面，替换 sk-your-deepseek-api-key
# 注意：DeepSeek API 是付费的，但非常便宜（约 1元/百万token）
DEEPSEEK_API_KEY=sk-你把这里替换成真实的key
DEEPSEEK_API_BASE=https://api.deepseek.com
DEEPSEEK_MODEL_NAME=deepseek-chat

# ========== 阿里云短信（暂时可留空） ==========
# 开发环境下不需要填，验证码会打印在终端日志中
# 部署到服务器正式使用时再配置
ALIBABA_SMS_ACCESS_KEY=
ALIBABA_SMS_SECRET=
ALIBABA_SMS_SIGN_NAME=
ALIBABA_SMS_TEMPLATE_CODE=

# ========== 上传文件目录 ==========
UPLOAD_DIR=./uploads
```

> **最小配置**：如果只是本地开发测试，你真正需要修改的只有两行：
> 1. `DATABASE_URL` 和 `DB_PASSWORD` — 改成你安装 PostgreSQL 时设的密码
> 2. `DEEPSEEK_API_KEY` — 改成你在 DeepSeek 平台获取的 key
>
> 其他的保持默认即可正常运行。

---

### 3.5 第四步：启动后端

```bash
# 进入后端目录
cd ta-platform/backend

# 创建虚拟环境（推荐，避免包冲突）
python -m venv venv

# 激活虚拟环境
# Windows CMD:
venv\Scripts\activate
# Windows PowerShell:
# .\venv\Scripts\Activate.ps1
# Git Bash / Linux / Mac:
# source venv/bin/activate

# 激活成功后，终端前面会显示 (venv) 字样

# 安装 Python 依赖包（首次运行需要，之后不用重复安装）
pip install -r requirements.txt
# 如果安装报错 Microsoft Visual C++，去 https://visualstudio.microsoft.com/visual-cpp-build-tools/
# 下载安装 "Build Tools for Visual Studio"，勾选 "C++ 生成工具" 即可

# 确认以上步骤都完成后，启动后端服务：
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

启动成功后，终端会显示：
```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Started reloader process
INFO:     Started server process
```

打开浏览器访问 **http://localhost:8000/api/docs**，如果能看到 API 文档页面（Swagger），说明后端启动成功！

> **常见启动报错**：
> - `could not connect to server` → PostgreSQL 没启动，去 Windows 服务里找到 `postgresql-x64-15` 并启动
> - `Error: DeepSeek API key is not configured` → .env 文件中的 DEEPSEEK_API_KEY 没填或填错了
> - `ModuleNotFoundError: No module named 'xxx'` → 依赖没装全，重新运行 `pip install -r requirements.txt`
> - `Address already in use` → 8000 端口被占用，换成 `--port 8001` 试试

### 3.6 第五步：启动前端

```bash
# 进入前端目录
cd ta-platform/frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端开发服务器运行在 http://localhost:5173，自动代理 `/api` 请求到后端 `:8000`。

### 3.7 可选：Docker 一键启动（无需手动安装数据库）

```bash
cd ta-platform

# 复制并编辑配置
cp .env.example .env
nano .env  # 填入 DEEPSEEK_API_KEY 和 JWT_SECRET

# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

启动后访问 http://localhost 即可使用。

## 4. 阿里云部署指南

### 4.1 服务器选购

推荐配置：
- **实例规格**：ecs.g7.large（2 vCPU / 4 GB 内存）
- **系统盘**：40 GB（高效云盘）
- **数据盘**：100 GB（存放上传文件、知识库）
- **带宽**：5 Mbps（按流量计费）
- **操作系统**：Ubuntu 22.04 LTS
- **地域**：选择离学校最近的（如 cn-shanghai、cn-beijing）

### 4.2 安全组配置

在阿里云控制台 → 安全组 → 添加规则：

| 端口 | 协议 | 来源 | 说明 |
|------|------|------|------|
| 22 | TCP | 你的IP | SSH 远程连接 |
| 80 | TCP | 0.0.0.0/0 | HTTP 网站访问 |
| 443 | TCP | 0.0.0.0/0 | HTTPS 加密访问 |

### 4.3 创建密钥对（购买时必做）

在阿里云 ECS 购买页面，会有一栏 **"密钥对"**，操作如下：

1. 点击 **"创建密钥对"** 按钮
2. 弹出窗口：
   - **名称**：填 `ta-platform-key`
   - **创建类型**：选 **"自动新建密钥对"**
3. 点击确定后，浏览器会**自动下载一个 `.pem` 文件**（如 `ta-platform-key.pem`）
4. **这个文件只下载一次，阿里云不帮你保存**，务必存到一个安全的地方，例如 `C:\Users\HP\.ssh\ta-platform-key.pem`

> 这个 `.pem` 文件就是你的服务器"钥匙"，丢了只能去控制台重置。

### 4.4 连接到服务器

**Windows 用户（推荐用 Git Bash）：**

如果你还没有 Git Bash，先去 https://git-scm.com 下载安装 Git，安装时会自带 Git Bash。

```bash
# 打开 Git Bash，用密钥文件连接服务器
ssh -i C:/Users/HP/.ssh/ta-platform-key.pem root@你的服务器公网IP
# 注意：路径用正斜杠 /
```

首次连接会提示 `Are you sure you want to continue connecting?`，输入 `yes` 回车。

**如果用的是 VS Code：**

安装 Remote-SSH 插件后，打开 `C:\Users\HP\.ssh\config` 文件（没有就新建），添加：

```
Host ta-server
    HostName 你的服务器公网IP
    User root
    IdentityFile C:/Users/HP/.ssh/ta-platform-key.pem
```

保存后，VS Code 左下角绿色按钮 → "连接到主机" → 选择 `ta-server`。

**Windows 终端 (CMD/PowerShell) 无法直接用 .pem 文件**，建议直接用 Git Bash 或 VS Code。

### 4.5 服务器初始化

```bash
# 连接上服务器后
ssh -i ~/.ssh/ta-platform-key.pem root@你的服务器公网IP
```

安装基础软件：

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | bash
apt install docker-compose -y

# 启动 Docker
systemctl enable docker
systemctl start docker

# 安装 Git
apt install git -y
```

### 4.6 部署项目

```bash
# 克隆项目到服务器
cd /opt
git clone 你的仓库地址 ta-platform
cd ta-platform

# 配置环境变量
cp .env.example .env
nano .env
```

**必须修改的 .env 配置：**

```ini
# 数据库密码（改成随机强密码）
DB_PASSWORD=你的数据库密码

# JWT 密钥（生成方式: openssl rand -hex 32）
JWT_SECRET=你生成的64位随机字符串

# DeepSeek API Key（从 platform.deepseek.com 获取）
DEEPSEEK_API_KEY=sk-你的key

# 阿里云短信（如需短信验证码功能）
ALIBABA_SMS_ACCESS_KEY=你的AccessKey
ALIBABA_SMS_SECRET=你的Secret
ALIBABA_SMS_SIGN_NAME=助教平台
ALIBABA_SMS_TEMPLATE_CODE=SMS_你的模板代码
```

启动服务：

```bash
# 构建并启动
docker-compose up -d

# 确认所有服务正常运行
docker-compose ps
# 应该看到 4 个服务都是 Up 状态：postgres, redis, backend, nginx
```

### 4.7 配置域名和 HTTPS

```bash
# 将域名解析到你的服务器 IP（在域名DNS管理后台添加 A 记录）

# 安装 Certbot
apt install certbot -y

# 获取 TLS 证书
certbot certonly --standalone -d 你的域名.com

# 证书路径
# 证书: /etc/letsencrypt/live/你的域名.com/fullchain.pem
# 私钥: /etc/letsencrypt/live/你的域名.com/privkey.pem

# 更新 Nginx 配置开启 HTTPS（将证书路径填入 nginx/nginx.conf）
# 重启 Nginx
docker-compose restart nginx
```

### 4.8 创建老师/助教账号

注册时默认创建的是学生账号。需要在数据库中手动设置角色：

```bash
# 进入 PostgreSQL
docker-compose exec postgres psql -U ta_admin -d ta_platform

# 将某个用户设置为老师
UPDATE users SET role = 'teacher' WHERE student_id = '教师工号';

# 将某个用户设置为助教
UPDATE users SET role = 'ta' WHERE student_id = '助教学号';

# 确认修改
SELECT id, student_id, display_name, role FROM users;

# 退出
\q
```

## 5. 三端使用说明

### 5.1 注册与登录

1. 打开网站，点击 "Register" 进入注册页
2. 填写：学号、手机号、密码、短信验证码
   - 在开发环境下，验证码会打印在**后端控制台日志**中
   - 生产环境配置阿里云短信后，验证码会通过短信发送
3. 注册成功后自动登录，跳转到对应角色页面
4. 之后使用学号或手机号 + 密码登录

### 5.2 学生端操作

**开始对话：**
1. 登录后进入学生主页面
2. 左侧边栏点击 "+" 开始新对话，或选择已有对话继续
3. 在底部输入框输入问题，按回车发送
4. AI 会基于知识库内容回答（如果选择了课程章节）

**选择课程章节问答：**
1. 左侧边栏展开课程（如 "Transistors & Optoelectronics"）
2. 点击具体章节（如 "Chapter 1"）
3. 右侧会显示 PPT 课件，左侧聊天区可针对该章节提问
4. AI 会优先从该章节的 TXT 知识库或 PDF 课件中查找答案

**上传文件：**
1. 点击输入框左侧的 📎 按钮
2. 选择 PDF / PPTX / DOCX / 图片文件
3. 输入问题，AI 会读取文件内容并回答

**语音输入（Chrome 浏览器）：**
1. 点击输入框中的 🎤 按钮
2. 允许浏览器使用麦克风
3. 说话后自动转为文字并发送

**主题切换：**
- 侧边栏底部点击 "Dark Mode" / "Light Mode" 切换

**PWA 安装（手机）：**
- iOS：Safari 打开网站 → 分享 → 添加到主屏幕
- Android：Chrome 打开网站 → 菜单 → 添加到主屏幕

### 5.3 老师端操作

1. 登录后进入老师管理面板
2. 左侧边栏**搜索学生**（按姓名或学号）
3. 点击学生，查看：
   - **使用统计**：消息数、对话数、登录次数、活跃时间
   - **对话列表**：所有历史对话，按时间排序
   - **对话详情**：点击某个对话，展开查看完整消息内容（含时间戳）
4. 页面底部查看**机器人状态**：
   - 在线状态（绿灯=工作中、黄灯=待机、灰灯=离线）
   - 电量百分比
   - 位置信息
   - 当日提问数量
   - 近期提问详情（含学生姓名和问答内容）

### 5.4 助教端操作

1. 登录后进入助教面板
2. 查看汇总数据卡片（学生数、消息总量、对话数、机器人提问数）
3. 助教**只能看到统计数据**，无法查看具体对话内容
4. 页面底部查看**机器人状态**（与老师端相同，但不显示提问详情）

## 6. 机器人接入说明

### 6.1 硬件要求

| 硬件 | 建议型号 |
|------|---------|
| 主控 | Raspberry Pi 4B（2GB+） |
| MCU | Arduino Mega / STM32F103 |
| 电机驱动 | L298N / TB6612 |
| 麦克风 | USB 麦克风 或 I2S MEMS 麦克风 |
| 扬声器 | 3.5mm 音箱 或 USB 音箱 |
| 电源 | 12V 锂电池组 + 5V 降压模块 |
| 底盘 | 两轮差速底盘 + 万向轮 |

### 6.2 机器人软件配置

```bash
# 在树莓派上
cd ta-platform/robot-client
pip install -r requirements.txt

# 配置连接参数
export ROBOT_SERVER_URL="ws://你的服务器IP:8000"
export ROBOT_ID="TA-Robot-01"
export ROBOT_AUTH_KEY="robot-secret-key-change-me"

# 启动机器人客户端
python main.py
```

### 6.3 通信协议

机器人每 2 秒向服务器上报：

```json
{
  "type": "telemetry",
  "battery": 85,
  "status": "active",
  "position_x": 1.5,
  "position_y": 3.2,
  "position_label": "图书馆二楼"
}
```

服务器向机器人下发：

```json
// 播放语音回答
{"type": "tts_response", "audio": "base64编码的音频...", "text": "这是回答内容"}

// 切换状态
{"type": "set_status", "status": "standby"}

// 移动指令（预留）
{"type": "move", "x": 2.0, "y": 4.0}
```

### 6.4 语音问答流程

```
学生说话 → 机器人麦克风采集 → WebSocket 发送音频到服务器
→ 服务器 ASR 语音识别 → DeepSeek AI 生成回答
→ 服务器 TTS 文字转语音 → WebSocket 发回机器人
→ 机器人扬声器播放
```

### 6.5 无硬件模拟模式

当树莓派不可用时，`telemetry.py` 和 `audio_capture.py` 会自动降级为模拟模式：
- 电量：缓慢下降，降到 5% 后自动复活为 100%
- 位置：保持在原点 (0, 0)
- 录音/播放：跳过实际操作，仅打印日志

## 7. 常见问题

### 注册时收不到短信验证码？

- 开发环境：验证码会打印在**后端终端日志**中，查看 `[DEV] SMS code for 手机号: 123456` 
- 生产环境：确认 `.env` 中已正确配置阿里云短信的 AccessKey、Secret、签名和模板代码

### DeepSeek API 调用失败？

- 检查 `.env` 中 `DEEPSEEK_API_KEY` 是否正确
- 检查服务器是否能访问外网（`curl https://api.deepseek.com`）
- 确认 DeepSeek 账户余额是否充足

### 前端页面空白？

- 确保后端服务正常运行（`docker-compose ps` 检查 backend 状态）
- 浏览器按 F12 查看 Network 面板，确认 API 请求是否成功
- 检查浏览器控制台是否有 JavaScript 报错

### 如何批量导入学生/课程？

```bash
# 进入 PostgreSQL
docker-compose exec postgres psql -U ta_admin -d ta_platform

# 批量创建学生账号（SQL 示例）
INSERT INTO users (id, student_id, phone, password_hash, display_name, role) VALUES
  (gen_random_uuid(), '2024001', '13800000001', '$2b$12$...', '张三', 'student'),
  (gen_random_uuid(), '2024002', '13800000002', '$2b$12$...', '李四', 'student');

# 批量创建课程
INSERT INTO courses (id, name, teacher_id, knowledge_base_path) VALUES
  (gen_random_uuid(), '半导体物理', '老师ID', 'Semiconductor Physics');
```

> 注意：`password_hash` 需要通过 Python 脚本生成 bcrypt 哈希，不能用明文。

### 如何更换课程/添加新课？

1. 在 `backend/app/knowledge_base/` 下新建课程文件夹，放入 PDF 和 TXT 知识库文件
2. 修改 `frontend/src/pages/StudentPage.tsx` 中的 `COURSES` 配置，添加新课程信息
3. 在数据库中 `courses` 表插入对应的课程记录
4. 重新构建前端：`cd frontend && npm run build`

### 如何更换 AI 模型？

修改 `.env` 中的模型名称：

```ini
# 使用 DeepSeek 的不同模型
DEEPSEEK_MODEL_NAME=deepseek-chat     # 默认
# DEEPSEEK_MODEL_NAME=deepseek-reasoner  # 推理增强
```
